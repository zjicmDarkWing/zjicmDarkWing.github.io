OpenPLC

openplc rce

  • OpenPLC 3 - Remote Code Execution (Authenticated) - Python webapps Exploit
    https://www.exploit-db.com/exploits/49803

然后运行exp,打到容器root：

user flag

信息

查看网卡可以发现一个wlan0，另外机器名本身也是和无线相关的：

plcrouter

所以后面就是看无线相关信息，扫描可以发现一个接入点：

WPS Pixie Dust Attack

后面就是基础的WPS Pixie Dust Attack获取PSK：

  • WPS Pixie Dust Attack (Offline WPS Attack)
    https://forums.kali.org/showthread.php?24286-WPS-Pixie-Dust-Attack-(Offline-WPS-Attack)
  • kimocoder/OneShot: Run WPS PIN attacks (Pixie Dust, online bruteforce, PIN prediction) without monitor mode with the wpa_supplicant
    https://github.com/kimocoder/OneShot

wifi

现在得到了密码就可以尝试连接wifi:

  • Using WPA_Supplicant to Connect to WPA2 Wi-fi from Terminal on Ubuntu 16.04 Server
    https://www.linuxbabe.com/command-line/ubuntu-server-16-04-wifi-wpa-supplicant

但连接后并没有自动分配ip，手动dhcp获取地址大概率崩溃：

  • linux - No DHCP lease with wlan0 - Super User
    https://superuser.com/questions/986239/no-dhcp-lease-with-wlan0

wlan service

所以需要修改service配置然后重启服务来获取地址:

  • Connecting to WiFi network using systemd and wpa-supplicant - SomLabs Wiki
    https://wiki.somlabs.com/index.php/Connecting_to_WiFi_network_using_systemd_and_wpa-supplicant

之后再查看，现在分配到了ip:

root flag
